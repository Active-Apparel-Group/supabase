# Supabase Tracking Documentation

## Purpose
This folder contains all documentation, migration scripts, and reference materials for the Supabase tracking schema and related project plans.

## Key Docs & Structure
- **overview/**: Project plans, phase summaries, status, and quick reference.
- **handoff/**: Frontend handoff and implementation reports.
- **migration-status/**: Migration status and template migration reports.
- **api-endpoints/**: API endpoint readiness and status.
- **migrations/**: SQL migration scripts for schema changes.
- **plans/**: Data requirements and folder/plan documentation.
- **ref/**: Technical reference files (HTML, JSON, etc.).
- **templates/**: Migration and timeline template guides.
- **docs/**: (Do not modify) Contains detailed research and blueprint documentation.

## How to Update Docs & Keep Tidy
- Consolidate related information into a single document when possible.
- Archive outdated docs in an `/archive` folder before deletion.
- Use clear, descriptive file names and folder organization.
- Update this README when adding or removing key docs.

## Doc Update Checklist for Contributors
1. Is your documentation unique, or does it duplicate existing info?
2. Is the doc placed in the correct subfolder?
3. Is the file name clear and descriptive?
4. Have you updated this README if you added/removed a key doc?
5. Is obsolete info archived or deleted?
6. Did you avoid modifying files in `/docs`?
7. Is your documentation concise and current?
